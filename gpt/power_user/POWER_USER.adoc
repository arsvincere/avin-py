= POWER USERS

== Workspace
Project = область жизни / большой проект
Chat    = кабинет / режим мышления / рабочий поток
File    = source of truth
Memory  = персонализация

== Memory
Не полагаются на память как на базу данных
Memory = привычки, стиль, предпочтения
Docs   = решения, архитектура, source of truth
Chat   = текущая работа

== HANDOFF
Сделай HANDOFF для Karl: кратко зафиксируй решения, scope и следующий шаг.

== Timestamp
Используют “дневные якоря”
## AVIN DAY START
date: 2026-07-01
focus: minimal DataSyncer -> Chart GUI

== Сильный запрос:
Контекст:
мы делаем AVIN, текущий фокус Chart -> GUI.

Состояние:
DataSyncPlan зелёный, DataSyncer ещё не начат.

Ограничение:
не углубляться в Data дальше minimal sync.

Задача:
дай план на 3 часа работы завтра.

== Decision log
После важного обсуждения:

Сделай decision log:
- decision
- why
- alternatives rejected
- consequences
- next step

Для AVIN это золото.

== Checkpoint
Во время долгой сессии:
Сделай checkpoint:
- что уже решили
- что сделано
- что осталось
- где риск уйти в сторону

Это помогает не проснуться через 5 часов в
DataSyncerStatusReportFormatterFactory.

== Stop rule
Очень полезно для тебя:

Сейчас проверь scope.
Мы всё ещё движемся к Chart -> GUI?
Или уже строим город вокруг сарая?

== Project files / sources
decision log
AVIN_CONTEXT.adoc
В моем случае - лучше вообще всю папку GPT

== Scheduled Tasks
Tasks подходят для напоминаний, recurring summaries и monitoring: OpenAI описывает их как способ создавать one-off/recurring tasks и проверки изменений с уведомлениями. Есть лимиты активных задач по плану, и задачи не могут выполняться чаще одного раза в час.

Для тебя возможные применения:

каждое утро: спросить план AVIN
каждый вечер: попросить summary дня
раз в неделю: собрать weekly review

Но я бы сначала наладил ручной ритуал DAY START / DAY SUMMARY.

== Самая взрослая схема для тебя
1. Office утром:
   DAY START, focus, ограничения.
2. Рабочий кабинет:
   Karl / Ava / Edge / Navi.
3. Внутри дня:
   checkpoint после больших решений.
4. При переходе между чатами:
   HANDOFF.
5. Вечером:
   summary дня в Office.
6. Важные решения:
   перенос в repo/docs.

== Главное отличие новичка от power user
Новичок использует ChatGPT как:
    умный ответчик
Сильный пользователь использует как:
    когнитивную операционную систему:
    память + документы + роли + рабочие протоколы + summaries + handoffs

Ты уже идёшь именно во второй вариант. Просто сейчас надо перестать
надеяться на неявный контекст и оформить workflow явно.
Тогда ChatGPT будет не “иногда помнит, иногда фантазирует”,
а нормальным рабочим инструментом.

== Закрывать длинные чаты
Сделай FINAL HANDOFF для нового _____:
    - что было сделано
    - какие решения приняты
    - source of truth
    - что не делать
    - следующий шаг
    - важные файлы/модули

== “Команда” может дать красивую дискуссию, но не решение - лечение:
Дай мнения Карла, Эджа, Авы, Нави.
Потом дай одно итоговое решение.
Потом дай один next action.

== Когда роль поплыла... быстрый фикс:
Использовать короткий reset внутри чата

Когда видишь, что ответ начал плыть, можно писать:

Это работает лучше, чем спорить с конкретным ответом.

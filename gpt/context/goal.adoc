= Цель - проп компания
Что это меняет уже сегодня. Очень конкретно. Делать надо так:
- Архитектура — чистая, но без фанатизма публичного API.
- Документация — для себя будущего и возможной команды, а не для случайных пользователей с GitHub.
- GUI — в первую очередь под твою скорость анализа, а не под “чтобы новичок понял”.
- Analyse — закрытый слой, без мыслей “как это потом красиво показать”.
- Данные — хранить серьезно, воспроизводимо, чтобы можно было проверять гипотезы.
- Backtest/replay — делать как лабораторный инструмент, а не как красивую демку.
- Лицензию MIT можно держать в заголовках, но не обязательно публиковать всё прямо сейчас.
*AVIN больше нельзя строить как мечту обо всем сразу.*
*AVIN 2026–2028: закрытая личная research/visual platform.*
*Сначала edge. Потом капитал. Потом люди. Потом фонд.*

Что это меняет
- Теперь AVIN надо строить не как “удобный терминал”, а как лабораторию для производства проверяемых торговых гипотез.
- GUI нужен не потому что “красиво”.
GUI нужен, чтобы быстрее видеть рынок.
- Footprint нужен не потому что “трейдерская фича”.
Footprint нужен, чтобы находить асимметрии.
- Analyse нужен не потому что “модуль по плану”.
Analyse нужен, чтобы превращать наблюдения в статистику.
- Backtester нужен не потому что “у всех есть”.
Backtester нужен, чтобы отделять edge от галлюцинаций.
- Data storage нужен не потому что “архитектурно правильно”.
Data storage нужен, чтобы исследования были воспроизводимы.

= All Instruments id (Iid) - information about instrument
storage format:
Информация об инструментах хранится по единой storage-схеме Schema.IID.
Все значения в этой схеме сохраняются как строки: dict[str, str] / pl.String.
Это намеренное решение, потому что разные поставщики данных присылают
metadata инструментов с разными именами полей и разными типами значений.

Provider-specific данные нормализуются до Schema.IID перед сохранением.
Storage не интерпретирует бизнес-смысл этих полей и не пытается частично
типизировать lot, step, exchange, category и т.д.

Типизация происходит выше, в domain-классе Iid:
он хранит raw string data и отдаёт typed properties при обращении.

== Public API
Если метод мутирует переданный объект, он по умолчанию возвращает None.
Если метод возвращает объект, он не должен незаметно мутировать переданный объект.

- Builder-классы -> возвращают новый объект, объект не мутируют.
- strict accessors -> возвращает объект или exception, без side effects.
- find_* -> soft-accessor, возвращает объект | None, без side effects.
- has_* -> bool, без side effects.
- load_* -> мутирует объект, возвращает None.
- build_* -> мутирует объект, возвращает None.
- ensure_* -> гарантирует наличие данных в объекте, возвращает None.

== Don't use in public API:
- get_or_build_* пока не используем, если нет сильной причины.
- try_get_* / get_opt_* в AVIN не используем; для optional используем find_*.

== Open field, accessor method, property
Открытое поле
-> только для тупых immutable value objects / dataclass.
@property
-> дешёвый read-only доступ без параметров и без побочных эффектов.
Метод accessor
-> если есть логика, возможная ошибка, параметр или торговый смысл.

== Accessor contract
1. Строгий accessor получает данные, которые обязаны существовать:
- либо возвращает полностью готовые данные;
- либо бросает исключение.
Никаких: None, пустых объектов, автозагрузки, ленивого построения.

2. Для мягкого optional-доступа используется find_*.

== Exceptions
Суффикс Error. Использовать его для всех пользовательских исключений проекта.

Все сообщения исключений начинаются с заглавной буквы и не заканчиваются точкой.
raise ValueError("Tick footprint 1000 is unavailable")
✅ Заглавная буква.
✅ Без точки в конце.
✅ Короткое сообщение.
✅ Сообщение отвечает на вопрос "что произошло?", а не "что делать?".

Хорошо:
Cluster is empty
Time footprint M1 is unavailable
Tick ts 100 < last ts 120
Invalid timeframe

Плохо:
Please load ticks first.
Cannot continue because no footprint exists.
You should build footprint before using it.


== Temporary elements
Во время разработки или дебагания, бывает удобно создать какую-ту переменную
или функцию. Для таких случаев использовать префиксы:
DEV_* - для хардкодед констант нужных на 5 минут пока делаешь какую то фичу
DBG_* - для отладочных констант

Переменные и функции соответственно помечаются:
dev_
dbg_

Так же может быть комментарий рядом с кодом:
# dbg
# dev
Что означает что под ним идет блок кода который был нужен для
отладки/разработки и теперь можно удалять.


== Docstrings
AVIN uses docstrings to document important public contracts, domain semantics,
invariants and non-obvious behavior.

Docstrings are required for:
* public domain classes;
* public functions and methods with non-trivial logic;
* methods with important edge cases;
* objects with invariants;
* APIs shared between `domain`, `service`, `gui` and `analyse`.

Docstrings are not required for:
* obvious dunder methods;
* simple read-only properties;
* trivial private helpers;
* code where the name and implementation are clearer than an additional text.

If a docstring is written, it must be bilingual:
* English first;
* empty line;
* `-- ru`;
* Russian version.

Important rule: AVIN does not require 100% docstring coverage.
Docstrings should clarify architecture and behavior, not create noise.

New code should follow this rule.
Existing code should be improved when the file is touched.

Examples:

----
def upsert(self, bar: Bar) -> None:
    """
    Insert or replace bar by timestamp.

    -- ru
    Вставляет бар в график или заменяет существующий бар
    с таким же timestamp.
    """
    ...

@property
def current(self) -> Bar | None:
    """
    Last bar in chart, may be unfinished in realtime.

    -- ru
    Последний бар в графике, в realtime может быть незавершенным.
    """
    ...
----

= Север
AVIN — закрытая исследовательская платформа для создания будущей проп-компании.
AVIN → edge → свои деньги → проп-компания → фонд.

= Текущая цель
Дойти до первого рыночного наблюдения внутри собственного GUI.
- Footprint в GUI из реальных данных.
- сырые тиковые графики

= Путь
→ Ticks
→ Footprint GUI
→ наблюдение
→ гипотеза
→ статистическая проверка
→ торговый setup
→ риск-модель
→ маленькие реальные сделки
→ стабильный edge
→ проп-компания.

= Критерий прогресса
Уровень	        Вопрос
✅Data	        Могу ли я загрузить реальные ticks?
✅Build	        Могу ли я построить footprint?
  Visual	    Могу ли я увидеть footprint в GUI?
  Observe	    Могу ли я заметить повторяющуюся ситуацию?
  Hypothesis	Могу ли я формализовать ее?
  Analyse	    Могу ли я проверить ее на истории?
  Trade	        Могу ли я превратить ее в setup?
  Money	        Могу ли я торговать это с положительным ожиданием?

= Боевой план до первых денег
== Этап 1. Визуальный рынок
Цель:
видеть реальные ticks / footprint в AVIN.

Состав:
Tick storage;
FootprintBuilder;
TimeFootprint;
GUI viewport;
отрисовка cluster/ladder/level;
минимальная навигация.

Результат:
“Я могу глазами изучать рынок в AVIN.”

== Этап 2. Наблюдения
Цель:
начать находить повторяющиеся ситуации.
Нужны:
удобный replay или хотя бы прокрутка истории;
возможность смотреть разные дни/инструменты;
заметки вручную;
скриншоты/метки;
список наблюдений.

Результат:
“У меня есть 10–30 конкретных рыночных идей, которые можно проверить.”

== Этап 3. Analyse
Цель:
проверить, есть ли в наблюдениях статистика.

Нужны:
простые research scripts;
выборки;
фильтры;
подсчет after-event движения;
MFE/MAE;
распределения;
сравнение с random/base case.

Результат:
“Я знаю, какие наблюдения мусор, а какие стоит копать.”

== Этап 4. Setup
Цель:
превратить наблюдение в торговое правило.

Нужны:
условия входа;
invalidation;
stop;
target;
risk/reward;
контекст рынка;
режимы, где setup запрещен.

Результат:
“Есть один setup, который можно торговать малым объемом.”

== Этап 5. Маленькие реальные деньги
Цель:
проверить не код, а себя + рынок + execution.

Нужны:
журнал сделок;
риск на сделку;
дневной лимит;
ручной/полуавтоматический режим;
пост-анализ;
статистика реальных решений.

Результат:
“Есть первый маленький положительный trading loop.”

== Этап 6. Prop foundation
Цель:
из личного edge сделать систему.

Нужны:
повторяемость;
риск-контроль;
операционные правила;
внутренний tooling;
масштабирование капитала;
позже — люди.

Результат:
“Это уже не просто я торгую, это зачаток проп-компании.”
